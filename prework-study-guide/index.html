HTML

  • The head element contains information about the webpage.
  • the head element can contain tags such as title, style, base, link, meta, and script.
  • these all are used for the structure of the site and this information is not visible to the user directly
  • The body element represents the visible content shown to the user.
  • the body can contain elements that are semantic or non-semantic.
  • semantic - the name of the element describes its content and the role it plays on a webpage.
  • non-semantic - elements that are less obvious in regards to the role they can play but are still useful in the presentation of information
  • body elements can include heading, paragraph, lists, images, line breaks, and anchors.

CSS

  • CSS can be implemented through inline, internal style sheet, or external style sheet.
  • inline is good for quick fixes but not ideal for whole projects
  • internal style sheets affect a whole page but are limited to that page
  • external style sheets can be linked in the head of a page from outside file and are ideal for most projects

Git

  • git status: checks what branch we are currently on
  • git checkout -b: creates a new branch and switches to it
  • git clone: clones a repo from github (use ssh hyperlink)
  • git pull: update local to match current version in the cloud
